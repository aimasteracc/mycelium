# Getting Support

Mycelium is pre-alpha. Support channels are intentionally lightweight.

## Where to Ask

| What you need | Where to go |
|---|---|
| Quick question, idea exchange | [GitHub Discussions](https://github.com/aimasteracc/mycelium/discussions) |
| Bug report | [Open an Issue](https://github.com/aimasteracc/mycelium/issues/new?template=bug_report.yml) |
| Feature request | [Open an Issue](https://github.com/aimasteracc/mycelium/issues/new?template=feature_request.yml) |
| Request a new language pack | [Pack request template](https://github.com/aimasteracc/mycelium/issues/new?template=language_pack_request.yml) |
| Propose a design change | [Submit an RFC](rfcs/README.md) |
| Security issue | [SECURITY.md](SECURITY.md) — **not** public Issues |
| Code of Conduct issue | Same private channel as SECURITY, with `[CoC]` prefix |
| Commercial / partnership | `aimasteracc <at> proton.me` |

## Response Time Expectations

We are a small team backed by AI automation. Typical:

- Triage label on new issue: **24 hours**
- First substantive response: **48 hours**
- Resolution for simple bugs: **7 days**
- Major features: **next milestone cycle (2 weeks)**

If a thread goes silent for more than two weeks, ping `@aimasteracc` and
we will get back on it.

## What We Cannot Provide

- Custom integrations (yet — post-v1.0 we will consider sponsored work)
- 24/7 SLA support (the Hive runs 24/7, but humans don't)
- Backports beyond the latest minor of the two most recent majors

## Stay in the Loop

- **Star the repo** to track releases
- **Watch releases** for changelog notifications
- **Follow [@aimasteracc](https://github.com/aimasteracc)** on GitHub
- **Subscribe to Discussions** for community updates

## TL;DR

v0.2 is the release where every Mycelium capability is reachable from **all three** surfaces — CLI, MCP, and Skill — and the marquee feature (Hyphae DSL) actually works through the CLI for the first time. The release theme is **"three surfaces, one truth"**.

If we ship one thing right, it's that a brand-new user can install Mycelium and reach the symbol graph from whichever surface matches their workflow within 5 minutes.

## Why v0.2 (motivation)

v0.1.0/v0.1.1 closed the publish loop but two findings landed immediately afterward:

1. **The marquee feature was unreachable.** ~~`mycelium query` is a placeholder~~ — **RESOLVED as of v0.1.13**: `mycelium query` is fully implemented (`crates/mycelium-cli/src/query.rs`), the MCP twin `mycelium_query` exists, and `skills/hyphae-query/SKILL.md` covers it. The original gap (Issue [#151](https://github.com/aimasteracc/mycelium/issues/151)) is closed. The remaining v0.2 work here is **end-to-end dogfood validation** (pass rate 8/8 CLI commands).
2. **Independent eval found systemic gaps.** glm5.1 tested all 88 MCP tools and surfaced three global failures: stdout pollution ([#150](https://github.com/aimasteracc/mycelium/issues/150)), edge_kind case divergence ([#152](https://github.com/aimasteracc/mycelium/issues/152)), and graph-algorithm timeouts on small graphs ([#153](https://github.com/aimasteracc/mycelium/issues/153)).

Plus the unavoidable governance work: the Three-Surface Rule (Charter §5.13) says we ship CLI + MCP + Skill or we don't ship. We currently have 88 MCP tools / 8 CLI commands / **10 Skills** (as of v0.1.12). **v0.2 closes the remaining gaps** (Skills INDEX.md CI gate, dogfood 8/8, RFC-0093 Phase 3 error model).

## Goals

1. **Three-Surface Rule is enforced.** All 88 capabilities live in CLI, MCP, AND at least one category Skill. `skills/INDEX.md` is mechanical and CI-gated.
2. **Hyphae is the headline.** `mycelium query "<selector>"` works end-to-end. README example runs. `mycelium-rcig-hyphae-query` is a real Skill that AI agents discover automatically.
3. **The eval bugs are gone.** All 5 dogfood/glm5.1 findings ship as fixes.
4. **Performance has a floor.** Charter §2 SLA table grows rows for graph-algorithm classes with concrete targets. No tool times out on the project Mycelium itself indexes.
5. **Distribution unlocks.** Skill bundle submitted to (at least) the Claude Code marketplace.

## Non-goals (v0.2)

- ❌ New languages. The 10 packs stay at 10.
- ❌ Cross-repo / multi-root indexing (planned for v0.3).
- ❌ Persistent reactive subscriptions for long-running agents (v0.3).
- ❌ IDE plugins. Skill bundles are this release's "easier integration"; IDE is v0.3+.
- ❌ Storage format changes. `.mycelium/index.rmp` schema stays compatible.

## Risks and mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| Hyphae query implementation reveals deeper hyphae-executor bugs | M | Time-boxed to v0.1.3; if execution layer needs RFC, descope to v0.3 and ship CLI as MCP-passthrough first. |
| Graph-algorithm fixes require architectural changes | M | If a tool genuinely is O(N²) it can ship with `time_budget_ms` + partial results envelope; not blocking. |
| Skill marketplace submission timeline opaque | H | Submit early in v0.1.5; if blocked, ship Skills as direct `git clone` install in README. |
| 88 capabilities don't fit 8 categories cleanly | L | Add a 9th. The 8-category list is from external eval; trust but verify in Sprint v0.1.3. |
| New language requests during the cycle | M | Defer all to v0.3 unless the founder unblocks. v0.2 is about the existing 10. |

## Open questions

1. **Per-Skill versioning?** Should `skills/code-navigation/` have its own version that progresses independently of the crate version? Defer to v0.3 RFC.
2. **Skill marketplace = Claude Code only?** Or also Cursor, Codex, others? Track as v0.3 distribution work.
3. **Should `mycelium-rcig-*` crate names be revisited?** Name was forced by squatters. If the squatted names get yanked, opening a transfer request might be worth a single-sprint sidequest. Track as v0.3.
